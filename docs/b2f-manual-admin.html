1. ภาพรวมระบบ B2F

B2F (Business to Factory) คือระบบสั่งซื้อสินค้าจากโรงงานผู้ผลิต (Maker) ผ่าน LINE Bot + LIFF

Flow หลัก

สร้าง PO → โรงงานยืนยัน → โรงงานส่งของ → Admin ตรวจรับ → จ่ายเงิน → ปิด PO

บทบาท

  • Admin (DINOCO) — สร้าง PO, ตรวจรับสินค้า, จ่ายเงิน, จัดการเครดิต
  • Maker (โรงงาน) — ยืนยัน/ปฏิเสธ PO, แจ้งส่งของ, ขอเลื่อนส่ง

3. สร้างใบสั่งซื้อ (PO)

วิธีสร้าง PO

1
เปิดหน้าสั่งซื้อ
เข้า Admin Dashboard → แท็บ B2F → กดปุ่ม "สั่งโรงงาน"
หรือเปิด LIFF จากปุ่มใน LINE Group
2
เลือกโรงงาน
เลือกโรงงานจากรายชื่อ แสดงชื่อ + สถานะเครดิต + วงเงิน
3
เลือกสินค้า
เลือกสินค้าจากแคตตาล็อกของโรงงาน กรอกจำนวน + ราคาทุน
กดเพิ่มลงตะกร้า ได้หลายรายการ
4
ตรวจสอบ & ยืนยัน
ตรวจรายการ + ยอดรวม → กด "ยืนยันสั่งซื้อ"
ระบบสร้าง PO + แจ้งโรงงานผ่าน LINE อัตโนมัติ
Tip: เมื่อสร้าง PO สำเร็จ ระบบจะส่ง Flex Message พร้อมรูปใบสั่งซื้อไปยังห้อง LINE ของโรงงาน + Admin Group

4. ตรวจรับสินค้า

เมื่อโรงงานแจ้งส่งของ

ระบบจะส่ง Flex แจ้ง Admin พร้อมปุ่ม "ตรวจรับสินค้า" หรือเข้าผ่าน PO Ticket View

1
เปิดหน้าตรวจรับ
กดปุ่ม "ตรวจรับสินค้า" จาก Flex Message หรือ PO Ticket
2
กรอกจำนวนที่รับ
แต่ละรายการแสดง: สั่ง | ส่งมา | รับแล้ว | คงเหลือ
กรอกจำนวนที่รับจริงในรอบนี้ (ไม่เกิน "คงเหลือ")
3
ตรวจ QC
เลือก "ผ่าน" หรือ "ไม่ผ่าน" สำหรับแต่ละรายการ
ถ้าไม่ผ่าน: กรอกจำนวนที่ reject + เหตุผล + ถ่ายรูป
4
กรอกชื่อผู้ตรวจรับ + หมายเหตุ (ถ้ามี)
5
กด "ยืนยันตรวจรับ"
ระบบบันทึก + คำนวณยอดค้างจ่าย + แจ้งโรงงานผ่าน LINE
รับบางส่วน: ถ้ารับไม่ครบ PO จะเป็นสถานะ "รับบางส่วน" — โรงงานส่งเพิ่มได้ แล้ว Admin ตรวจรับรอบต่อไป

ตัวเลขสำคัญ

ตัวเลขความหมาย
สั่งจำนวนที่สั่งซื้อ
ส่งมาจำนวนที่โรงงานส่งมาแล้วทั้งหมด (สะสม)
รับแล้วจำนวนที่ผ่าน QC (ไม่นับ reject)
ไม่ผ่านจำนวนที่ QC ไม่ผ่าน
คงเหลือส่งมา - (รับแล้ว + ไม่ผ่าน) = ของที่ยังไม่ได้ตรวจ QC

5. จัดการ QC ไม่ผ่าน

เมื่อตรวจรับแล้วมีรายการ QC ไม่ผ่าน ระบบจะส่ง Flex "QC ไม่ผ่าน" แจ้งทั้ง Maker + Admin

3 ทางเลือกจัดการ

ทางเลือกรายละเอียดPO จะเป็น
ส่งทดแทน (Reship) โรงงานส่งของใหม่มาแทนที่ reject
PO กลับเป็น "รอผลิต" → โรงงานส่งอีกรอบ		 รอผลิต
สั่งใหม่ (Replacement) สร้าง PO ใหม่เฉพาะ SKU ที่ reject
PO เดิมปิดตามปกติ		 PO ใหม่ แบบร่าง
ตัดยอด (Write-off) ลดจำนวนสั่งให้เท่ากับที่รับผ่านจริง
ไม่ต้องส่งเพิ่ม		 รับของครบ
สำคัญ: ของที่ reject ไปแล้วจะไม่กลับมาให้รับซ้ำ — ถ้าโรงงานส่งทดแทน จะนับเป็นของชิ้นใหม่ที่เพิ่มจำนวน shipped

6. จ่ายเงิน

เงื่อนไข

  • จ่ายได้เมื่อ PO สถานะ รับของครบ หรือ ชำระบางส่วน
  • ยอดค้างจ่าย = มูลค่าที่รับจริง (ไม่ใช่ยอดสั่ง)
  • จ่ายเกินยอดค้างไม่ได้ (ระบบป้องกัน)
1
เปิด PO Ticket → เลื่อนลงส่วน "บันทึกจ่ายเงิน"
2
กรอก: จำนวนเงิน, วิธีจ่าย (โอน/เช็ค/เงินสด), วันที่, เลขอ้างอิง
แนบสลิปได้ (ถ้ามี)
3
กด "บันทึก"
- จ่ายครบ: PO → ชำระครบ → ปิด PO อัตโนมัติ
- จ่ายบางส่วน: PO → ชำระบางส่วน
ป้องกันจ่ายซ้ำ: ระบบมี lock ป้องกันการกดซ้ำ + duplicate detection ภายใน 10 วินาที

7. ระบบเครดิตโรงงาน

หลักการ

  • ยอดค้างจ่ายเพิ่ม เมื่อ Admin ตรวจรับสินค้า (ไม่ใช่ตอนสร้าง PO)
  • ยอดค้างจ่ายลด เมื่อ Admin จ่ายเงิน
  • คิดเฉพาะของที่ผ่าน QC เท่านั้น (reject ไม่นับ)

Credit Hold

ประเภทเงื่อนไขปลดล็อก
Auto Hold ยอดค้าง ≥ วงเงินเครดิต จ่ายเงินจน < วงเงิน → ปลดอัตโนมัติ
Manual Hold Admin กดล็อกเอง Admin ต้องปลดเอง (ไม่ auto-unhold)
เมื่อ Credit Hold: ไม่สามารถสร้าง PO ใหม่ให้โรงงานนี้ได้ จนกว่าจะปลด hold

8. ยกเลิก / แก้ไข PO

ยกเลิก PO

  • ยกเลิกได้ทุกสถานะ ยกเว้น "ปิด PO" และ "ยกเลิก"
  • ต้องระบุเหตุผล
  • ถ้ามีตรวจรับไปบ้างแล้ว: ระบบจะคำนวณยอดค้างจ่ายใหม่อัตโนมัติ

แก้ไข PO (Amend)

  • แก้ไขได้เมื่อสถานะ: รอยืนยัน, รอผลิต, ปฏิเสธ
  • แก้ได้: จำนวน, ราคา, เพิ่ม/ลบรายการ
  • หลังแก้ → PO จะส่งให้โรงงานยืนยันใหม่
  • ระบบนับจำนวนครั้งที่แก้ไข (amendment count)

9. Admin Dashboard

3 แท็บหลัก

แท็บ 1: รายการสั่งซื้อ (Orders)

  • KPI: รอยืนยัน | รอผลิต | เลยกำหนด | ค้างจ่าย
  • ตารางรายการ PO พร้อม filter ตามสถานะ
  • ค้นหาตาม PO number, ชื่อโรงงาน
  • ปุ่ม quick action: ดูรายละเอียด, ตรวจรับ, จ่ายเงิน

แท็บ 2: โรงงาน (Makers)

  • รายชื่อโรงงาน + สถานะเครดิต
  • ยอดค้างจ่าย + วงเงิน + เครดิตคงเหลือ
  • ประวัติ PO ล่าสุดของแต่ละโรงงาน
  • จัดการ: Hold/Release เครดิต

แท็บ 3: เครดิต (Credit)

  • ยอดรวมค้างจ่ายทั้งหมด
  • รายละเอียดค้างจ่ายต่อโรงงาน
  • ประวัติการจ่ายเงิน
  • Audit trail ทุกรายการ

11. คำถามที่พบบ่อย

ถาม: โรงงานส่งของบางส่วนได้ไหม?

ได้ โรงงานสามารถแจ้งส่งของหลายรอบ ระบบจะสะสมจำนวน shipped จนกว่าจะครบ

ถาม: ตรวจรับบางรายการก่อนได้ไหม?

ได้ PO จะเป็นสถานะ "รับบางส่วน" จนกว่าจะรับครบ

ถาม: ถ้า reject แล้วโรงงานส่งใหม่ ระบบนับยังไง?

ของที่ reject ไม่กลับมาให้รับซ้ำ โรงงานส่งใหม่จะเพิ่ม shipped ขึ้น แล้ว Admin ตรวจรับของใหม่ที่ส่งมา

ถาม: จ่ายเงินหลายรอบได้ไหม?

ได้ จ่ายบางส่วนก่อน แล้วจ่ายที่เหลือทีหลัง

ถาม: ยอดค้างจ่ายคิดยังไง?

มูลค่าของที่รับผ่าน QC จริง (ไม่ใช่ยอดสั่ง) ลบด้วยยอดที่จ่ายแล้ว

ถาม: ยกเลิก PO ที่รับของไปบ้างแล้วได้ไหม?

ได้ ระบบจะคำนวณยอดค้างจ่ายใหม่อัตโนมัติ (ตัดส่วนที่ยกเลิกออก)
