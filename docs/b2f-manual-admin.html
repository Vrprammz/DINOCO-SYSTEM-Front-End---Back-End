1. ภาพรวมระบบ

B2F (Business to Factory) คือระบบสั่งซื้อสินค้าจากโรงงานผู้ผลิต (Maker) โดยสั่งผ่าน LINE + LIFF ทำทุกอย่างในแอป LINE ได้เลย

ขั้นตอนหลัก

  1. Admin สร้างใบสั่งซื้อ → ระบบแจ้งโรงงานผ่าน LINE
  2. โรงงานยืนยัน + แจ้งวันส่ง
  3. โรงงานแจ้งส่งของ → ระบบแจ้ง Admin
  4. Admin ตรวจรับ + QC → ระบบคำนวณยอดค้างจ่ายอัตโนมัติ
  5. Admin โอนเงินให้โรงงาน + ส่งสลิปลงห้อง LINE → ระบบตัดยอดอัตโนมัติ

4. ตรวจรับสินค้า

เมื่อโรงงานแจ้งส่งของ ระบบจะส่งการ์ดแจ้งเตือนพร้อมปุ่ม "ตรวจรับสินค้า"

1
กดปุ่ม "ตรวจรับสินค้า" จากการ์ดแจ้งเตือน หรือเปิดจากหน้ารายละเอียดใบสั่งซื้อ
2
กรอกจำนวนที่รับจริง
แต่ละรายการแสดง: สั่ง | ส่งมา | รับแล้ว | คงเหลือ
กรอกจำนวนรอบนี้ (ไม่เกิน "คงเหลือ")
3
ตรวจคุณภาพ (QC)
เลือก "ผ่าน" หรือ "ไม่ผ่าน" แต่ละรายการ
ถ้าไม่ผ่าน: กรอกจำนวนที่ไม่ผ่าน + เหตุผล + ถ่ายรูปประกอบ
4
กรอกชื่อผู้ตรวจรับ + หมายเหตุ (ถ้ามี) → กด "ยืนยันตรวจรับ"
รับบางส่วน: ถ้ารับไม่ครบ สถานะจะเป็น "รับบางส่วน" — โรงงานส่งเพิ่มได้ แล้วตรวจรับรอบต่อไป

ความหมายตัวเลข

ตัวเลขความหมาย
สั่งจำนวนที่สั่งซื้อ
ส่งมาจำนวนที่โรงงานส่งมาแล้วทั้งหมด (สะสมทุกรอบ)
รับแล้วจำนวนที่ผ่านการตรวจคุณภาพ (ไม่นับของที่ไม่ผ่าน)
ไม่ผ่านจำนวนที่ตรวจคุณภาพไม่ผ่าน
คงเหลือส่งมา − ผ่านตรวจทั้งหมด (ทั้งผ่าน+ไม่ผ่าน) = ของที่ยังไม่ได้ตรวจ

6. จ่ายเงินโรงงาน

ขั้นตอน

1
ตรวจสอบยอดค้างจ่าย
เปิดรายละเอียดใบสั่งซื้อ → ดูยอด "มูลค่าที่รับจริง" และ "ค้างจ่าย"
ยอดค้างจ่าย = มูลค่าที่รับผ่าน QC จริง (ไม่ใช่ยอดสั่ง)
2
โอนเงินให้โรงงาน
โอนผ่านธนาคารตามเลขบัญชีของโรงงาน
3
บันทึกการจ่ายเงินในระบบ
เปิดรายละเอียดใบสั่งซื้อ → กรอก:
• จำนวนเงินที่โอน
• วิธีจ่าย (โอน / เช็ค / เงินสด)
• วันที่โอน + เลขอ้างอิง
• แนบรูปสลิป
4
ระบบตัดยอดอัตโนมัติ
• จ่ายครบ → สถานะเปลี่ยนเป็น ชำระครบ → ปิดใบสั่งซื้อ อัตโนมัติ
• จ่ายบางส่วน → สถานะ ชำระบางส่วน จ่ายที่เหลือทีหลังได้
• ระบบส่งการ์ดแจ้งโรงงานว่าได้รับเงินแล้ว

7. ระบบเครดิตโรงงาน

หลักการ

  • ยอดค้างจ่ายเพิ่ม เมื่อ ตรวจรับสินค้า (ไม่ใช่ตอนสร้างใบสั่งซื้อ)
  • ยอดค้างจ่ายลด เมื่อ โอนเงินให้โรงงาน
  • คิดเฉพาะของที่ผ่าน QC เท่านั้น (ของที่ไม่ผ่านไม่นับ)

ระงับเครดิต (Credit Hold)

ประเภทเงื่อนไขปลดล็อก
อัตโนมัติยอดค้าง ≥ วงเงินจ่ายเงินจน < วงเงิน → ปลดเอง
ล็อกเองAdmin กดล็อกAdmin ต้องปลดเอง
เมื่อถูกระงับ: ไม่สามารถสร้างใบสั่งซื้อใหม่ให้โรงงานนี้ได้ จนกว่าจะปลด

8. ยกเลิก / แก้ไข

ยกเลิกใบสั่งซื้อ

  • ยกเลิกได้ทุกสถานะ ยกเว้น ตั้งแต่ "โรงงานจัดส่ง" ขึ้นไป (ส่งของแล้ว ยกเลิกไม่ได้)
  • ต้องระบุเหตุผล
  • เลือกยกเลิกทีละรายการ หรือ ติ๊กเลือกหลายรายการ แล้วยกเลิกพร้อมกัน
  • ระบบส่งการ์ดแจ้งโรงงานว่ายกเลิกพร้อมเหตุผล

แก้ไขใบสั่งซื้อ

  • แก้ได้เมื่อสถานะ: รอยืนยัน, รอผลิต, ปฏิเสธ
  • แก้จำนวน, ราคา, เพิ่ม/ลบรายการได้
  • หลังแก้ → ระบบส่งให้โรงงานยืนยันใหม่

9. หน้าจัดการ (Dashboard)

แท็บ 1: รายการสั่งซื้อ

  • ตัวเลขสรุป: รอยืนยัน | ใกล้ส่ง | เลยกำหนด | ค้างจ่าย
  • กรองตามสถานะ + ค้นหาตามเลขที่ / ชื่อโรงงาน
  • ติ๊กเลือกหลายรายการ → ยกเลิกพร้อมกัน

แท็บ 2: โรงงาน

  • รายชื่อโรงงาน + ยอดค้างจ่าย + วงเงิน
  • ระงับ/ปลดเครดิต

แท็บ 3: เครดิต

  • ยอดรวมค้างจ่ายทั้งหมด + ประวัติการจ่ายเงิน

10. การ์ดแจ้งเตือนทาง LINE

ทุกเหตุการณ์สำคัญ ระบบจะส่งการ์ดแจ้งเตือนไปห้อง LINE อัตโนมัติ:

เหตุการณ์ส่งไปที่เนื้อหา
สร้างใบสั่งซื้อโรงงาน + Adminรายการสินค้า + ปุ่มยืนยัน/ปฏิเสธ
โรงงานยืนยันAdminยืนยันแล้ว + วันส่ง
โรงงานปฏิเสธAdminปฏิเสธ + เหตุผล
โรงงานส่งของโรงงาน + Adminตาราง สั่ง/ส่ง/เหลือ/รอบนี้ + ปุ่มตรวจรับ
ตรวจรับสินค้าโรงงาน + Adminตาราง สั่ง/รับ/เหลือ/รอบนี้ + ยอดค้างจ่าย
QC ไม่ผ่านโรงงาน + Adminรายการที่ไม่ผ่าน + เหตุผล + รูป
จ่ายเงินโรงงานจำนวน + วิธีจ่าย + สลิป
ปิดใบสั่งซื้อโรงงาน + Adminสรุปเสร็จสิ้น
ยกเลิกโรงงาน + Adminแจ้งยกเลิก + เหตุผล

11. ถาม-ตอบ

โรงงานส่งของบางส่วนได้ไหม?

ได้ ส่งได้หลายรอบจนกว่าจะครบ

ตรวจรับบางรายการก่อนได้ไหม?

ได้ สถานะจะเป็น "รับบางส่วน" จนกว่าจะรับครบ

ถ้าของไม่ผ่าน QC แล้วโรงงานส่งใหม่ ระบบนับยังไง?

ของที่ไม่ผ่าน QC จะไม่กลับมาให้รับซ้ำ โรงงานส่งใหม่จะนับเป็นของชิ้นใหม่

จ่ายเงินหลายรอบได้ไหม?

ได้ จ่ายบางส่วนก่อน แล้วจ่ายที่เหลือทีหลัง

ยอดค้างจ่ายคิดยังไง?

มูลค่าที่รับผ่าน QC จริง (ไม่ใช่ยอดสั่ง) ลบด้วยยอดที่จ่ายแล้ว

ยกเลิกใบสั่งซื้อที่รับของไปบ้างแล้วได้ไหม?

ยกเลิกได้เฉพาะก่อนโรงงานส่งของ หลังส่งแล้วยกเลิกไม่ได้
