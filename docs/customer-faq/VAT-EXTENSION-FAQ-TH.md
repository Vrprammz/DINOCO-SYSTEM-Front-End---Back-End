# คำถามที่พบบ่อย — การต่ออายุประกัน + ใบกำกับภาษี

> สำหรับลูกค้า DINOCO ที่ต่ออายุประกันผ่าน LINE LIFF Marketplace
> ปรับปรุงล่าสุด: 2026-05-18 · ระบบ VAT Compliance V.1.0

ใช้สำหรับ CS Team อ้างอิงตอบลูกค้า Facebook / LINE ฯลฯ — ไม่ต้องการตอบ verbatim ถ้าลูกค้าไม่ถาม

---

## A. คำถามทั่วไป

### A1. "ทำไมต่อประกันถึงมี VAT 7%?"

ตอบ:
> DINOCO เป็นผู้จดทะเบียนภาษีมูลค่าเพิ่ม (VAT registered) ในนามบริษัท พีพีที กรุ๊ป คอร์ปอเรชั่น จำกัด (Tax ID: 0105564033573) การต่ออายุประกันถือเป็นบริการที่ต้องเสีย VAT 7% ตามกฎหมาย ลูกค้าจะได้รับใบกำกับภาษี/ใบเสร็จรับเงินที่สมบูรณ์เพื่อใช้ลดหย่อนภาษี (ถ้ามี) หรือเก็บเป็นหลักฐานการชำระเงิน

### A2. "ราคาที่เห็น ${X} บาท รวม VAT แล้วหรือยัง?"

ตอบ:
> ราคาที่แสดงใน LIFF Marketplace **รวม VAT 7% แล้ว** (VAT-inclusive) ลูกค้าจะเห็นแถว "VAT 7%" แยกในหน้ายืนยันคำสั่งซื้อเพื่อความโปร่งใส แต่ยอดที่ต้องจ่ายจริงคือ "รวมทั้งสิ้น" ที่แสดงด้านล่าง

### A3. "ทำไมต่อ 4 ชิ้นถึงได้ใบกำกับฉบับเดียว?"

ตอบ:
> สินค้าของท่านเป็น **ชุด (SET)** — เช่นชุดกันล้ม ชุด Top Box ฯลฯ ระบบจะต่ออายุประกันทุกชิ้นในชุดพร้อมกันโดยอัตโนมัติ (4 เพลทขยายอายุประกันเท่ากัน) และออกใบกำกับ 1 ใบรวม รายการในใบจะระบุ "ต่ออายุประกัน 1 ปี — ชุด NX500 IRONB (ชุด 4 ชิ้น)" — นี่คือนโยบายมาตรฐาน เพราะเป็นการขายแพ็คเกจ 1 ครั้ง ลูกค้าจ่ายเงินครั้งเดียวก็ต้องได้ใบกำกับครั้งเดียว

## B. หลังจ่ายเงิน

### B1. "ทำไมยังไม่ได้รับใบกำกับใน LINE?"

ตอบ:
> ระบบจะส่งใบกำกับให้ภายใน 1-2 นาทีหลังแอดมินตรวจสอบสลิปแล้ว ถ้าครบ 5 นาทีแล้วยังไม่ได้รับ กรุณาตรวจสอบ:
> 1. ท่าน **ได้ block บอท DINOCO** หรือยกเลิกการเป็นเพื่อนหรือไม่
> 2. ถ้าใช่ — ปลดบล็อก/แอดเพื่อนใหม่ + แจ้ง Facebook Admin ขอ resend
> 3. ถ้าไม่ — แจ้ง Facebook Admin พร้อมหมายเลข payment_ref (EXT-XXXX-...) เพื่อ resend manual

### B2. "ดาวน์โหลดใบกำกับฉบับเต็มได้ที่ไหน?"

ตอบ:
> กดปุ่ม **"📄 ดูใบกำกับภาษีฉบับเต็ม"** ที่ด้านล่างของ Flex card ที่ได้รับใน LINE → จะเปิดหน้า HTML ใน browser → กดปุ่ม Print หรือ Save as PDF
>
> หรือเข้า DINOCO Member Dashboard → คำสั่งซื้อของฉัน → ค้นหา payment_ref → ดูใบกำกับ

### B3. "ตอนแอดมินขอตรวจสอบสลิป จะใช้เวลานานแค่ไหน?"

ตอบ:
> ตามมาตรฐาน 2-4 ชั่วโมงในเวลาทำการ (09:00-18:00 จันทร์-เสาร์) ถ้าจ่ายนอกเวลาทำการอาจจะรอข้ามวัน เมื่อแอดมินตรวจสอบเสร็จและประกันต่ออายุแล้ว ระบบจะส่งใบกำกับมาให้อัตโนมัติทันที

## C. ปัญหาในใบกำกับ

### C1. "ชื่อใน VAT receipt ผิด — แก้ได้ไหม?"

ตอบ:
> ระบบใช้ชื่อตามที่ลงทะเบียนในระบบ DINOCO ถ้าต้องการแก้:
> 1. **แก้ชื่อในระบบก่อน** ผ่าน DINOCO Member Dashboard → โปรไฟล์ → แก้ไข
> 2. **แจ้ง Facebook Admin** ขอ resend ใบกำกับ พร้อมแนบ payment_ref
> 3. แอดมินจะใช้ endpoint `/resend` เพื่อสร้างใบกำกับใหม่ที่มีชื่อที่อัพเดทแล้ว
>
> หมายเหตุ: ใบกำกับเดิมจะยังอยู่ในระบบเพื่อเป็น audit trail แต่ลูกค้าจะได้รับใบใหม่ที่ถูกต้อง

### C2. "ที่อยู่บนใบกำกับเป็นที่อยู่ DINOCO ไม่ใช่ที่อยู่ผมเอง?"

ตอบ:
> ใบกำกับภาษีมาตรฐานจะระบุ **ที่อยู่ของผู้ขาย** (DINOCO PPT Group) ในส่วนหัวเป็นข้อมูลทางภาษีตามที่กรมสรรพากรกำหนด ส่วน **ชื่อผู้ซื้อ** จะอยู่ในแถว "ลูกค้า" ที่อยู่ลูกค้าไม่ได้ระบุในใบกำกับเพราะการต่ออายุประกันใช้ Tax ID ของลูกค้า (ถ้าจดทะเบียน) เป็นข้อมูลแยก

### C3. "ทำใบกำกับให้ในนามบริษัทได้ไหม? (ใช้ลดหย่อนภาษีบริษัท)"

ตอบ:
> ได้ ลูกค้าต้องลงทะเบียนข้อมูลบริษัทใน DINOCO Member Dashboard ก่อนต่อประกัน:
> 1. โปรไฟล์ → ข้อมูลภาษี
> 2. กรอก ชื่อบริษัท + Tax ID 13 หลัก + ที่อยู่บริษัท
> 3. ทำการต่อประกัน → ใบกำกับจะออกในนามบริษัทอัตโนมัติ
>
> หมายเหตุ: ต้องลงทะเบียนข้อมูลบริษัท **ก่อน** ต่อประกัน ถ้าต่อแล้วเพิ่งมาลง ต้องแจ้ง Facebook Admin ขอ re-issue

## D. การคืนเงิน

### D1. "ขอคืนเงินได้ไหม?"

ตอบ:
> สามารถขอคืนเงินได้ภายใน 7 วันหลังต่ออายุ และ **ยังไม่ได้ส่งเคลม** เลย กรณีต่อผิดเช่น ผิดแผน ผิดเลขเพลท ฯลฯ
>
> ขั้นตอน:
> 1. แจ้ง Facebook Admin พร้อม payment_ref + เหตุผล
> 2. แอดมินตรวจสอบ — ถ้าผ่านเงื่อนไข จะเริ่มกระบวนการคืน
> 3. คืนเงินผ่านช่องทางเดียวกับที่จ่ายมา (PromptPay → บัญชีต้นทาง)
> 4. ระยะเวลาคืน: 3-5 วันทำการ
>
> **สำคัญ**: ใบกำกับภาษีฉบับเดิมจะ **ไม่ถูกยกเลิก** ในระบบ (audit trail) แต่จะมีการออกใบลดหนี้ (Credit Note) ให้ลูกค้าเป็นหลักฐานการคืนเงิน

### D2. "คืนเงินแล้วประกันยังต่ออยู่ไหม?"

ตอบ:
> ไม่ ระบบจะ **คืนค่า warranty_until** ของเพลทกลับเป็นวันเดิมก่อนต่อ ลูกค้าจะกลับสู่สถานะประกันเดิม ถ้าเพลทอยู่ในชุด (SET) ระบบจะคืนทุกเพลทในชุดพร้อมกัน

## E. นโยบายข้อมูล (PDPA)

### E1. "ข้อมูลในใบกำกับปลอดภัยไหม?"

ตอบ:
> ปลอดภัย ไฟล์ PNG ของใบกำกับเก็บในระบบ DINOCO และเข้าถึงได้เฉพาะ:
> - **เจ้าของบัญชี** (ลูกค้าที่จ่ายเงิน — ตรวจสอบผ่าน LINE login)
> - **แอดมิน** (สำหรับการสนับสนุนลูกค้า)
>
> URL ของไฟล์มี **HMAC token 24 ตัวอักษร** (~120 บิตของ entropy) ทำให้ผู้อื่นไม่สามารถเดา URL ของท่านได้แม้รู้หมายเลขใบเสร็จ ระบบป้องกัน enumeration attack ด้วย rate limiting + .htaccess + REST 404 collapse

### E2. "ขอลบข้อมูลทั้งหมดของฉันได้ไหม?"

ตอบ:
> ตามสิทธิ์ภายใต้ PDPA ลูกค้าสามารถขอลบข้อมูลส่วนตัวได้ผ่าน GDPR Data Requests แต่:
> - ข้อมูลใบกำกับภาษีต้อง **เก็บ 5 ปี** ตามกฎหมายภาษี (กรมสรรพากร อาจตรวจสอบย้อนหลัง)
> - หลัง 5 ปี ระบบจะ anonymize อัตโนมัติ (ชื่อ + ที่อยู่ → null, เก็บเฉพาะหมายเลข + ยอด)
> - ลูกค้าที่ต้องการลบทันที — แจ้ง Facebook Admin + ลงนามเอกสารยินยอมว่า DINOCO จะไม่สามารถออกใบกำกับซ้ำให้ได้ในอนาคต

## F. ปัญหา operational

### F1. "ระบบขึ้น 'ระบบ VAT ปิดชั่วคราว' — เกิดอะไรขึ้น?"

ตอบ:
> ระบบ VAT มี **kill switch** ที่แอดมินสามารถปิดได้ทันที (instant) ในกรณีต้อง investigate ปัญหา หรือ update ข้อมูลทางภาษี ระหว่างที่ปิด:
> - **ลูกค้าใหม่** ที่จ่ายเงินจะยังได้ประกัน (warranty extend ปกติ) แต่ **ไม่ได้รับใบกำกับ** จนกว่าจะเปิดระบบใหม่
> - **ลูกค้าเก่า** ที่จ่ายเงินก่อนปิด ยังเข้าถึงใบกำกับเดิมได้ปกติ
>
> เมื่อเปิดระบบใหม่ แอดมินจะ resend ใบกำกับให้ลูกค้าที่ค้างไว้ — ไม่ต้องทำอะไร เพียงรอ Flex ใหม่จาก LINE

### F2. "GET 404 บน URL ใบกำกับ — ใบหายไปไหน?"

ตอบ:
> สาเหตุที่เป็นไปได้:
> 1. **HMAC token ไม่ถูกต้อง** — URL ของท่านอาจ corrupt ตอน copy/paste → ขอใหม่ผ่าน Member Dashboard
> 2. **แอดมิน resend ใบกำกับ** — URL เดิมอาจถูก regenerate (ป้องกัน old URL leak) → ตรวจ Flex ใหม่ใน LINE
> 3. **ระบบจริงๆ ไม่พบใบ** — กรณีนี้แจ้ง Facebook Admin

## G. CS Team Internal Reference

### G1. ค้นหา extension จาก payment_ref

```sql
SELECT id, sn, customer_name, price_paid, payment_status, paid_at
FROM wp_dinoco_sn_warranty_extensions
WHERE payment_ref = 'EXT-XXX-YYY-ZZZ';
```

### G2. Resend ใบกำกับ (admin only)

```
POST /wp-json/dinoco-vat/v1/resend/{id}
Cookie: WP admin session
```

หรือผ่าน Admin Dashboard → การเงิน → VAT รายเดือน → ค้นหา → ปุ่ม "ส่งใหม่"

### G3. ตรวจสอบสถานะ VAT system

```
GET /wp-json/dinoco-marketplace-tools/v1/diagnose
Cookie: WP admin session

ดูค่า:
  - vat_master_enabled: '1' (ON) / '0' (OFF)
  - vat_is_ready: true (tax_id + company + address ครบ)
  - vat_is_active: true (master AND ready)
```
